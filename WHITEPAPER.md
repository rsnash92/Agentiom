# Agentiom: The Complete Platform for Stateful AI Agents

## Vision Document & Product Roadmap

---

## Executive Summary

**Agentiom is the Vercel for AI agents.**

We solve the fundamental problem that has plagued AI agent development: **agents forget everything**.

Every existing solution either:
- Forces developers to manage complex infrastructure (databases, vector stores, orchestration)
- Charges for always-on compute even when agents are idle
- Treats memory as an afterthought, bolted on via external services

Agentiom takes a different approach: **persistent memory as a first-class primitive**.

```
One command. Persistent memory. $0 when idle.

$ agentiom deploy
```

---

## The Problem

### The Stateless Trap

Modern AI agents are built on stateless infrastructure. Every interaction starts from zero:

```
Conversation 1: "My name is Rob, I work at Acme Corp"
Conversation 2: "Who am I?" â†’ "I don't know"
```

Developers waste hours:
- Re-explaining context every session
- Building custom persistence layers
- Managing databases, vector stores, and caches
- Paying for 24/7 compute for agents that work 1% of the time

### The Infrastructure Tax

To build a stateful agent today, developers need:

| Component | What You Manage |
|-----------|-----------------|
| Compute | VPS, containers, serverless functions |
| Database | Postgres, Redis, or SQLite |
| Vector Store | Pinecone, Weaviate, or pgvector |
| Orchestration | Message queues, job runners |
| Monitoring | Logs, metrics, alerts |
| Scaling | Load balancers, replicas |

**The infrastructure tax kills agent projects before they ship.**

### The Cost Problem

Always-on agents are expensive:

| Platform | Cost per Agent | Idle Cost |
|----------|----------------|-----------|
| Railway | ~$5-20/month | Same |
| Render | ~$7-25/month | Same |
| EC2 | ~$15-50/month | Same |
| **Agentiom** | ~$0.50-5/month | **$0** |

## The Solution

### Core Thesis

**Agents need their own computers.**

Not metaphorically. Literally:
- Persistent filesystem that survives restarts
- Isolated runtime per agent
- Memory that accumulates over time
- Compute that sleeps when not needed

## Product Architecture

### The Developer Experience

```bash
# Install
npm install -g @agentiom/cli

# Create agent
agentiom init my-agent
cd my-agent

# Write your agent
code src/index.ts

# Deploy
agentiom deploy

# Done. Agent is live with persistent memory.
```

### The SDK

```typescript
import { Agent } from '@agentiom/sdk';

const agent = new Agent({ name: 'assistant' });

// Handle different triggers
agent.on('webhook', async (event, ctx) => {
  // Simple state - key/value, instant
  const count = ctx.state.increment('visits');

  // Semantic memory - searchable, persistent
  await ctx.memory.add({
    text: `User asked: ${event.body.message}`,
    type: 'conversation'
  });

  // Search past memories
  const relevant = await ctx.memory.search({
    query: event.body.message,
    limit: 5
  });

  // Files - persistent filesystem
  await ctx.files.write('logs/request.json', event.body);

  // LLM - built-in, with memory context
  const response = await ctx.llm.chat({
    messages: [
      { role: 'system', content: `Relevant memories:\n${relevant.map(m => m.text).join('\n')}` },
      { role: 'user', content: event.body.message }
    ]
  });

  return { message: response, visits: count };
});

agent.on('slack', async (event, ctx) => {
  // Same ctx.state, ctx.memory, ctx.files, ctx.llm
  // Different trigger source
});

agent.on('cron', '0 9 * * *', async (ctx) => {
  // Daily task at 9am
  const report = await generateDailyReport(ctx);
  await ctx.integrations.slack.send('#reports', report);
});

agent.on('email', async (email, ctx) => {
  // Inbound email processing
  await ctx.memory.add({ text: email.body, type: 'email' });
});

export default agent;
```
